# 🌿 Green Shopping Extension

A Chrome extension that scores products for sustainability in real time while you shop online — powered by AI analysis, a Node.js backend, and a PostgreSQL database.

---

## 🚀 Live Demo

- **API:** https://green-shopping-extension.onrender.com
- **Admin Dashboard:** https://green-shopping-extension.onrender.com/admin.html

---

## ✨ Features

### Chrome Extension
- 🟢 **Real-time sustainability scoring** — AI-powered 100-point score on every product page
- 🧮 **5-category breakdown** — Materials, Certifications, Packaging, Brand Ethics, Durability
- ⚠️ **Greenwashing detection** — flags vague claims like "eco-friendly" with no certification backing
- 🌱 **Greener alternatives** — suggests better products with scores and direct links
- ⚖️ **Side-by-side comparison** — compare current product vs greener alternative
- 🔖 **Save products** — bookmark products you like via chrome.storage
- 🎯 **User priorities** — set personal filters (Vegan, Plastic-free, Carbon Neutral, Fair Trade, Cruelty-free)
- 💬 **Feedback system** — rate score accuracy directly from the sidebar
- 💡 **AI improvement suggestions** — Claude-powered tips to find better alternatives

## 🏃 Running Locally

### Prerequisites
- Node.js v18+
- PostgreSQL
- A free [Groq API key](https://console.groq.com)

### 1. Clone the repo
```bash
git clone https://github.com/YOUR_USERNAME/green-shopping-extension.git
cd green-shopping-extension
```

### 2. Set up the backend
```bash
cd backend
npm install
```

Create a `.env` file in `backend/`:
```
PORT=3000
DB_HOST=localhost
DB_PORT=5432
DB_NAME=green_shopping
DB_USER=postgres
DB_PASSWORD=your_password
GROQ_API_KEY=gsk_your_key_here
```

### 3. Set up the database
Create the database in PostgreSQL:
```sql
CREATE DATABASE green_shopping;
```

Then run the schema (tables for brands, products, certifications, feedback, admin_users) and seed data from the project wiki.

### 4. Start the backend
```bash
node index.js
# Server running on http://localhost:3000
```

### 5. Build the extension bundle
```bash
cd ../extension
npm install
npm run build
# content.bundle.js generated
```

### 6. Load into Chrome
1. Open `chrome://extensions`
2. Enable **Developer mode**
3. Click **Load unpacked**
4. Select the `extension/` folder

The backend is deployed on **Render** (free tier).

### Re-deploying after changes
1. Push changes to GitHub — Render auto-deploys on every push to `main`
2. Rebuild the extension bundle: `npm run build`
3. Reload the extension at `chrome://extensions`

### Note on free tier
Render's free tier spins down after inactivity. The first request after a period of inactivity may take 50+ seconds to respond. Subsequent requests are fast.

## 🤖 How AI Scoring Works

1. Extension extracts product title, brand, description from the page
2. Sends data to `/score` endpoint on the backend
3. Backend sends it to Groq's `llama-3.3-70b-versatile` model with a structured prompt
4. AI returns a scored JSON object with per-category reasoning
5. If AI fails, falls back to keyword-based scoring automatically
6. Score is displayed in the sidebar with full breakdown and reasoning

## 📊 Scoring Categories

| Category | Max Score | What it checks |
|----------|-----------|----------------|
| Materials | 30 | Organic, recycled, bamboo, biodegradable |
| Certifications | 25 | GOTS, Fair Trade, B Corp, FSC, OEKO-TEX |
| Packaging | 20 | Plastic-free, compostable, recyclable |
| Brand Ethics | 15 | Carbon neutral, ethical manufacturing |
| Durability | 10 | Lifetime warranty, repairable, reusable |

**Greenwashing detection:** If vague terms like "eco-friendly" or "green" appear without any certification, a warning flag is shown.

## 🔧 Development Notes

### Rebuilding after JS changes
Every time you edit any `.js` file in the extension folder, rebuild:
```bash
cd extension
npm run build
```
Then reload the extension at `chrome://extensions`.

### Adding new product categories
1. Add scoring rules to `backend/categories.js`
2. Add detection regex to `extension/categoryDetector.js`
3. Seed alternative products for the new category in the database

### Adding new supported sites
1. Create `extension/sites/newsite.js` with extraction logic
2. Register it in `extension/sites/index.js`
3. Add the URL pattern to `manifest.json` under `content_scripts.matches` and `host_permissions`
4. Rebuild the bundle

## 📝 API Reference

### `GET /score`
**Query params:** `title`, `description`, `brand`, `category`

**Response:**
```json
{
  "score": 74,
  "source": "ai",
  "breakdown": {
    "materials":      { "score": 28, "max": 30, "reasoning": "Bamboo handles..." },
    "certifications": { "score": 0,  "max": 25, "reasoning": "No certifications mentioned" },
    "packaging":      { "score": 18, "max": 20, "reasoning": "Plastic-free packaging" },
    "brand":          { "score": 13, "max": 15, "reasoning": "Family-owned brand..." },
    "durability":     { "score": 10, "max": 10, "reasoning": "Designed to last..." }
  },
  "greenwashingDetected": false,
  "greenwashingFlags": [],
  "summary": "A sustainable option with eco-friendly materials...",
  "improvements": ["Obtain FSC certification", "Clarify packaging recyclability"]
}
```

```json
      "price": "$6",
      "url": "https://amazon.com/..."
    }
  ]
}
```
